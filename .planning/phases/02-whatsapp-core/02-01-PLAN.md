## 1. DB Schema Extensions

### New Tables

#### `contacts`
```sql
CREATE TABLE contacts (
  id text PRIMARY KEY,                    -- nanoid
  organization_id text NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  phone_number text NOT NULL,             -- E.164 format: +1234567890
  display_name text,                      -- WhatsApp display name
  created_at timestamp DEFAULT now() NOT NULL,
  updated_at timestamp DEFAULT now() NOT NULL,
  UNIQUE(organization_id, phone_number)   -- one contact per number per org
);
```

#### `conversations`
```sql
CREATE TABLE conversations (
  id text PRIMARY KEY,                    -- nanoid
  organization_id text NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  contact_id text NOT NULL REFERENCES contacts(id) ON DELETE CASCADE,
  assigned_user_id text REFERENCES users(id) ON DELETE SET NULL,
  status text NOT NULL DEFAULT 'active',  -- active | expired | closed
  window_expires_at timestamp,            -- 24h from last inbound message
  last_message_at timestamp,
  created_at timestamp DEFAULT now() NOT NULL,
  updated_at timestamp DEFAULT now() NOT NULL
);
```

#### `messages`
```sql
CREATE TABLE messages (
  id text PRIMARY KEY,                    -- nanoid
  conversation_id text NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
  organization_id text NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  direction text NOT NULL,               -- inbound | outbound
  wa_message_id text,                    -- Meta message ID (wamid.xxx)
  content text,                          -- message text
  status text NOT NULL DEFAULT 'sent',   -- sent | delivered | read | failed
  sent_at timestamp,
  created_at timestamp DEFAULT now() NOT NULL,
  UNIQUE(wa_message_id)                  -- idempotency on webhook replays
);
```

### RLS Policies
All three tables use `organization_id` column for tenant isolation, same pattern as existing tables.

### Indexes
- `contacts(organization_id)`, `contacts(phone_number)`
- `conversations(organization_id)`, `conversations(contact_id)`, `conversations(assigned_user_id)`
- `messages(conversation_id)`, `messages(organization_id)`, `messages(wa_message_id)`

## 4. Implementation Details

### Webhook Handler (GET - verification)
```
GET /api/webhooks/whatsapp
Query: hub.mode=subscribe, hub.verify_token=<token>, hub.challenge=<number>
Response: hub.challenge value (plain text 200)
```

### Webhook Handler (POST - receive messages)
```
POST /api/webhooks/whatsapp
Body: Meta webhook payload
1. Validate X-Hub-Signature-256 header (HMAC-SHA256 of raw body with WHATSAPP_API_TOKEN)
2. Return 200 immediately
3. Process each entry.changes[].value:
   - If messages[]: handle inbound message
     a. Find org by phoneNumberId (whatsapp_phone_number_id on organizations)
     b. Upsert contact (phone + org, set display_name from profile.name)
     c. Upsert conversation (find by contact+org, create if not exists)
     d. Set windowExpiresAt = now + 24h
     e. Insert message record
   - If statuses[]: handle status update
     a. Find message by wa_message_id
     b. Update status field
```

### Send Message API
```
POST /api/whatsapp/send
Body: { conversationId: string, content: string }
Auth: Clerk (getCurrentUser)
1. Get conversation + org credentials
2. Check window not expired (windowExpiresAt > now)
3. POST to https://graph.facebook.com/v21.0/{phoneNumberId}/messages
   Body: { messaging_product: "whatsapp", to: phoneNumber, type: "text", text: { body: content } }
4. Insert message record with waMessageId from response
5. Return message object
```

### 24h Window Status
Computed field on conversation fetch:
```typescript
windowStatus: conversation.windowExpiresAt && new Date(conversation.windowExpiresAt) > new Date()
  ? "active" : "expired"
```

### Supabase Realtime
Use @supabase/supabase-js client with anon key on client side:
- Subscribe to `messages` table INSERT filtered by `conversation_id = eq.{id}`
- Subscribe to `conversations` table UPDATE filtered by `organization_id = eq.{orgId}`

## 5. Meta Cloud API Integration Notes

- API version: v21.0
- Endpoint: `https://graph.facebook.com/v21.0/{PHONE_NUMBER_ID}/messages`
- Auth header: `Authorization: Bearer {WHATSAPP_API_TOKEN}`
- Webhook signature: `X-Hub-Signature-256: sha256={hmac}` using app secret (or token)
- Message ID in response: `messages[0].id` (wamid.xxx format)
