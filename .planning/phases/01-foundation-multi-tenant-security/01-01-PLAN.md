    **userRoleEnum**: pgEnum with values `['admin', 'manager', 'agent']`

    **organizations** table:
    - `id`: text, primary key (stores Clerk organization ID like `org_xxx`)
    - `name`: text, not null
    - `slug`: text, not null, unique
    - `whatsappPhoneNumberId`: text, nullable (Meta phone number ID, added for TENANT-06)
    - `whatsappBusinessAccountId`: text, nullable (Meta WABA ID)
    - `createdAt`: timestamp, defaultNow, not null
    - `updatedAt`: timestamp, defaultNow, not null

    **users** table:
    - `id`: text, primary key (stores Clerk user ID like `user_xxx`)
    - `organizationId`: text, not null, references organizations.id with onDelete cascade
    - `email`: text, not null
    - `firstName`: text, nullable
    - `lastName`: text, nullable
    - `imageUrl`: text, nullable
    - `role`: userRoleEnum, not null, default 'agent'
    - `isActive`: boolean, default true, not null
    - `createdAt`: timestamp, defaultNow, not null
    - `updatedAt`: timestamp, defaultNow, not null

    Add indexes:
    - `users_org_id_idx` on users.organizationId
    - `users_email_idx` on users.email

    Export all tables, enums, and inferred TypeScript types (using `$inferSelect` and `$inferInsert` from Drizzle).

    Create `src/lib/db/index.ts`:
    - Import `drizzle` from `drizzle-orm/postgres-js` and `postgres` from `postgres`
    - Create postgres client with `prepare: false` and `max: 1` (CRITICAL for Supabase transaction pooling)
    - Export `db` (drizzle instance with schema for relational queries)
    - Export `setTenantContext(organizationId: string)` function that executes `SET LOCAL app.current_organization_id = '${organizationId}'` using a parameterized query via sql template tag to prevent SQL injection
    - Export `setUserContext(userId: string, role: string)` function that sets `app.current_user_id` and `app.user_role` session variables

    Run `npx drizzle-kit generate` to generate the initial migration SQL.

    After Drizzle generates the migration, create a SECOND manual migration file `0001_enable_rls.sql` containing:

    ```sql
    -- Enable RLS on all multi-tenant tables
    ALTER TABLE organizations ENABLE ROW LEVEL SECURITY;
    ALTER TABLE users ENABLE ROW LEVEL SECURITY;

    -- Organizations: tenant can only see their own org
    CREATE POLICY org_tenant_isolation ON organizations
      FOR ALL
      USING (id = current_setting('app.current_organization_id', true)::text);

    -- Users: tenant can only see users in their org
    CREATE POLICY users_tenant_isolation ON users
      FOR ALL
      USING (organization_id = current_setting('app.current_organization_id', true)::text);

    -- Users INSERT: must match current org context
    CREATE POLICY users_insert_policy ON users
      FOR INSERT
      WITH CHECK (organization_id = current_setting('app.current_organization_id', true)::text);
    ```

    IMPORTANT: Use `current_setting('app.current_organization_id', true)` with `true` as second arg so it returns NULL instead of erroring when the variable is not set. This prevents crashes for service-role connections that bypass RLS.

    Do NOT run `drizzle-kit migrate` yet - the user needs to set up their Supabase project first and provide DATABASE_URL. The migrations are generated and ready to apply.

    Add npm scripts to package.json:
    - `"db:generate": "drizzle-kit generate"`
    - `"db:migrate": "drizzle-kit migrate"`
    - `"db:studio": "drizzle-kit studio"`
