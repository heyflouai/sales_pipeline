  <action>
    **Clerk webhook handler** (`src/app/api/webhooks/clerk/route.ts`):
    Create POST endpoint that:
    1. Reads `CLERK_WEBHOOK_SECRET` from env
    2. Extracts `svix-id`, `svix-timestamp`, `svix-signature` headers
    3. Verifies signature using `Webhook` from `svix` package (see RESEARCH.md Pitfall 2)
    4. Returns 400 if missing headers or invalid signature
    5. Handles these webhook event types:
       - `organization.created`: Insert into `organizations` table (id, name, slug)
       - `organization.updated`: Update organization name/slug
       - `organizationMembership.created`: Insert into `users` table (id from public_user_data.user_id, organizationId from organization.id, email, firstName, lastName, role mapped from Clerk role)
       - `organizationMembership.updated`: Update user role
       - `organizationMembership.deleted`: Set user isActive to false (soft delete)
       - `user.updated`: Update user email/name fields
       - `user.deleted`: Hard delete user record
    6. Role mapping: Clerk `org:admin` -> DB `admin`, `org:manager` -> DB `manager`, everything else -> DB `agent`
    7. Use `eq()` from drizzle-orm for WHERE conditions
    8. IMPORTANT: Webhook handler must use a separate DB connection that bypasses RLS (service role). Create `src/lib/db/admin.ts` that exports an `adminDb` instance connecting WITHOUT setting tenant context. Use this for webhook operations since webhooks don't have a tenant session.
    9. Return 200 on success with JSON body `{ received: true }`
    10. Wrap in try/catch, return 500 with error message on failure

    **RBAC utilities** (`src/lib/auth/rbac.ts`):
    Create role-checking functions:
    - `requireRole(role: 'org:admin' | 'org:manager' | 'org:agent')`: Gets orgRole from `auth()`, checks role hierarchy (admin=3 > manager=2 > agent=1), throws 403 error if insufficient
    - `hasRole(role)`: Returns boolean instead of throwing
    - `isAdmin()`: Shorthand for `hasRole('org:admin')`
    - `canManageUsers()`: Returns true if admin or manager
    - `getCurrentUser()`: Returns `{ userId, orgId, orgRole }` or throws

    **Dashboard layout** (`src/app/(dashboard)/layout.tsx`):
    Create a sidebar layout:
    - Left sidebar (w-64) with navigation links: Conversations, Settings
    - Top bar with `<OrganizationSwitcher />` from `@clerk/nextjs` and `<UserButton />` from `@clerk/nextjs`
    - Main content area that renders children
    - Use Tailwind for styling: bg-gray-50 for main area, white sidebar, border-r divider
    - `OrganizationSwitcher` should have `afterSelectOrganizationUrl="/conversations"` and `afterCreateOrganizationUrl="/conversations"`
    - Sidebar should show active state based on current pathname using `usePathname()` from `next/navigation`
    - NOTE: Since usePathname is a client hook, extract the sidebar nav into a `"use client"` component (`src/components/layout/sidebar-nav.tsx`)
